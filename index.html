Surge iOS 配置与分流订阅

当前配置采用 WireGuard HongKong 作为主节点， 策略组包含 Proxy、AI、🇭🇰 HK、🇺🇸 US。 分流规则覆盖广告拦截、中国大陆直连、Apple、AI、Microsoft、GitHub 与兜底代理。

安全提醒：当前配置中包含 WireGuard 私钥、PSK、服务器 IP 与 API 密码，公开仓库前必须替换或移除。

配置入口 Surge iOS

推荐在 Surge iOS 中直接导入完整配置文件。

代理策略 Proxy Group

Proxy = select, 🇭🇰 HK, 🇺🇸 US
AI = select, 🇺🇸 US, Proxy

🇭🇰 HK = url-test, policy-regex-filter=港|HK|Hong, interval=600, tolerance=50
🇺🇸 US = url-test, policy-regex-filter=美|US|States, interval=600, tolerance=50

AI 默认优先走美国节点，其余流量默认进入 Proxy 手动选择组。

RULE-SET,https://jkoch14.me/rules/OpenAI.list,AI

GEOIP,CN,DIRECT
FINAL,Proxy,dns-failed
已修正：原配置中 Apple.list,DIRECR 拼写错误，应为 DIRECT。

DNS 与网络优化 General

  • DNS 使用 system、腾讯 DNS、阿里 DNS。
  • 劫持 Google / Cloudflare 公共 DNS，避免应用绕过 Surge。
  • 启用 UDP 优先与代理 UDP 测试。
  • LAN 网段、Tailscale、银行、认证页走直连。
dns-server = system,119.29.29.29,223.5.5.5
proxy-test-url = http://cp.cloudflare.com/generate_204
proxy-test-udp = cp.cloudflare.com@1.1.1.1
block-quic = all-proxy
udp-priority = true

已发现问题 Fix Required

  • DIRECR 应改为 DIRECT。
  • direct = Direct 与策略组中的 DIRECT 不是同一个概念，建议删除该代理项。
  • external-controller-access 与 http-api 暴露到 0.0.0.0，公网环境存在风险。
  • WireGuard 私钥、PSK、endpoint 不应出现在公开页面或公开仓库。
